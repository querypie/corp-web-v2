# Overview

QueryPie의 Database Access Control에서 발생하는 로그들을 Database Logs에서 확인할 수 있습니다.
DB 접속 권한 부여와 회수, DB 접근 이력, 수행한 쿼리, DML 쿼리 스냅샷 등의 정보를 확인할 수 있습니다.
그 외, QueryPie 내 DB 접속 패스워드 실패 및 장기미접속으로 인한 커넥션 잠금 내역을 조회하고 또한 해제할 수 있습니다.

# QueryPie DAC에서 제공하는 로그

## DB Access History

- **DB 커넥션 연결 및 연결해제** 이력을 기록합니다.

## Query Audit

- DB 작업 실행이 끝난 **쿼리 및 액션 수행 이력**을 기록합니다.

## Running Queries

- 현재 실행 중인 쿼리를 조회할 수 있습니다.
  쿼리 실행이 종료되면 Query Audit에 내용이 이력으로 기록됩니다.

## DML Snapshots

> **사전 조건:** 해당 커넥션의 DML Snapshot 옵션이 활성화되어있어야 합니다.

- 쿼리파이에서 **INSERT / UPDATE / DELETE** 쿼리 실행 완료시 전후 데이터를 기록합니다.

## Account Lock History

> **사전 조건:** 해당 커넥션의 **Maximum Login Failures, Specified Time Interval Before Lockout** 정책을 따릅니다.

- QueryPie의 사용자별 **커넥션 패스워드 실패 및 장기미접속으로 인한 커넥션 잠금 이력**을 표시합니다.
- 패스워드 오입력, 비인가 IP의 접속 시도 등 접속 실패한 횟수를 모두 카운트 하며, 관리자가 설정한 접속 실패 임계치 도달시 접속 계정이 Lock 상태가 됩니다.

## Access Control Logs

- QueryPie 사용자/그룹에 부여되는 **DB 커넥션에 대한 접근 권한 부여 및 회수 이력**을 기록합니다.
- DB 커넥션 권한의 부여 및 회수는 Access Control 메뉴에서 처리됩니다.
